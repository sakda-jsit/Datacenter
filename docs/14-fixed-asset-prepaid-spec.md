# 14 Fixed Asset & Prepaid Spec (Requirement v11)

> ที่มา: workbook sheets `FA`, `SUM`, `PREPAID`, `PREPAID ANTIVIRUS` + คำตอบ User 2026-06-04

## Fixed Asset Register (FA)
`FA` = ทะเบียนสินทรัพย์หลัก (source of truth) ไม่ใช่ working paper

### ค่าเสื่อมราคา 2 ชุด
- ต้องคำนวณและเก็บแยก **ชุดบัญชี** และ **ชุดภาษี** — ห้ามรวมยอดโดยไม่มีประเภทกำกับ
- รายงานต้องระบุชัดว่าใช้ยอดชุดใด
- สูตรหลัก (Excel): `DATEDIF` (อายุถือครอง), `ROUND`, ค่าเสื่อมสะสม, มูลค่าสุทธิ = ราคาทุน − ค่าเสื่อมสะสม

### อัตรา/อายุค่าเสื่อม (คำตอบ #4)
- **มี master มาตรฐานอัตรา/อายุต่อประเภทสินทรัพย์** (`AssetTypeMaster`)
- ระบบ default จาก master เมื่อสร้างสินทรัพย์ และ **override รายตัวได้**

### Lifecycle: ซื้อ / จำหน่าย / ขาย / ตัดจำหน่าย
- รองรับสถานะ Active / Disposed / Sold / WrittenOff
- กระทบ: ราคาทุน, ค่าเสื่อมสะสม, ค่าเสื่อมงวด, มูลค่าสุทธิ, ยอดสรุป `SUM` → NOTE2
- **ขายสินทรัพย์ (คำตอบ #3): ระบบคำนวณกำไร/ขาดทุนอัตโนมัติ**
  ```
  กำไร/ขาดทุนจากการขาย = ราคาขาย − มูลค่าสุทธิ ณ วันขาย
  ```

### Validation
- ต้องมีข้อมูลครบก่อนคำนวณค่าเสื่อม: รายการ, วันที่ซื้อ, ราคาทุน, อัตรา, ประเภท
- disposal ต้องมี: วันที่, ประเภทเหตุการณ์, สินทรัพย์, ราคาทุน+ค่าเสื่อมสะสมก่อนรายการ, เอกสาร/เหตุผล

### Express-owned field lock (สอดคล้องหลักการ "ห้าม CRUD ทับข้อมูล Express")
สินทรัพย์ที่นำเข้าจาก FAMAS (`IsFromExpress = true`) ฟิลด์ที่ Express เป็นเจ้าของ
(รหัส/ชื่อ/ราคาทุน/วันได้มา/มูลค่าซาก/ยอดยกมา/หมวด) เป็น **read-only** ในหน้าแก้ไข + บังคับฝั่ง server
(แก้ที่ Express แล้วนำเข้าใหม่ — Express = source of truth). แก้ผ่านระบบได้เฉพาะฟิลด์ที่ app เป็นเจ้าของ:
อัตราค่าเสื่อม (override ชุดบัญชี/ภาษี), บัญชี GL ที่ผูก, สถานะ/การจำหน่าย, หมายเหตุ — การ re-import จะไม่ทับ
อัตราที่ override ไว้. สินทรัพย์ที่ป้อนเอง (บริษัทที่ FAMAS ว่าง, `IsFromExpress = false`) แก้ได้ทุกฟิลด์.

### รายงาน (RPT-013)
รายการ, ประเภท, วันที่ซื้อ, ราคาทุน, ค่าเสื่อมบัญชี, ค่าเสื่อมภาษี, ค่าเสื่อมสะสม, มูลค่าสุทธิ,
รายการจำหน่าย/ขาย/ตัดจำหน่าย, สรุปตามประเภท (เชื่อม NOTE2)

## Prepaid Expense Schedule (PREPAID)
- ทุกรายการใช้ **pattern ตัดจ่ายเดียวเป็น rule กลาง** (คำตอบยืนยัน A-024)
- กระจายยอดตามวันที่เริ่ม–สิ้นสุด (เดือนแรก prorate ตามวัน เช่น `C/12 × days/31`)
- Validation: วันเริ่ม ≤ วันสิ้นสุด; ยอดตัดจ่ายรวม ≤ มูลค่าตั้งต้น; คงเหลือไม่ติดลบ; คงเหลือ = ตั้งต้น − ตัดจ่ายสะสม
- หากพบรายการไม่ตรง pattern ระหว่าง migration → แยกเป็น exception ถาม User (ห้ามเดา)

### รายงาน (RPT-014)
รายละเอียด, เอกสารอ้างอิง, มูลค่าตั้งต้น, วันเริ่ม/สิ้นสุด, ตัดจ่ายงวด, ตัดจ่ายสะสม, คงเหลือ
