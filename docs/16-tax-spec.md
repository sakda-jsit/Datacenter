## 1. TAX — ภาษีเงินได้นิติบุคคล
- **ระบบคำนวณทั้งหมด** (A-017) จาก TB + รายการบวกกลับ/หักออก + รายการปรับปรุงภาษี
- ต่อยอดจากโมดูล **ภ.ง.ด.50 ที่เสร็จแล้ว** (X4 + WHT → counterpart ในงบดุล TXP/TXR)
- แยกข้อมูลต้นทาง / parameter / ผลลัพธ์ ให้ตรวจสอบได้; ข้อมูลไม่ครบต้องเตือนก่อนสรุป
- ผลขาดทุนสะสมยกไป (sheet `ผลขาดทุน`): `E = prev + B − C − D`

## 2. PP.30 — VAT รายเดือน
- ดึง Input/Output VAT อัตโนมัติ (A-018) — ไม่ให้ key ยอดสรุปเอง
```
ภาษีสุทธิ = ภาษีขาย (OUTPUT VAT) − ภาษีซื้อ (INPUT VAT)
balance สะสม = prev + ภาษีที่เกิด − จำนวนที่ตัดชำระ
```
- แยกเดือน: `MONTH(date)` / `MID(docno,8,2)`
- Validation: ยอด PP.30 ต้องตรง Input/Output VAT รายเดือน; ไม่ตรงแสดงรายการต่าง
- **บล็อก DBF**: รอโครงสร้างตาราง Input/Output VAT จริงของ Express

## 3. PND.3 / PND.53 — Flow upload PDF + reconcile
ขั้นตอน (A-019):
1. ฝ่ายบัญชี upload PDF ที่ download จากสรรพากร
2. ระบบ extract ข้อมูลด้วย **template parser** (คำตอบ #12: layout คงที่)
3. reconcile กับข้อมูล WHT/Express
4. ตรง → เก็บเข้าระบบ; ไม่ตรง → แสดงรายการต่าง ไม่บันทึก final อัตโนมัติ
5. เก็บไฟล์ PDF ต้นฉบับเป็นหลักฐาน

### Field ที่ต้อง extract จาก PDF
ประเภทแบบ, เลขผู้เสียภาษี, เดือน/ปีภาษี, ประเภทยื่น (ปกติ/เพิ่มเติม) + ครั้งที่, จำนวนราย,
รวมยอดเงินได้, รวมยอดภาษีนำส่ง, เงินเพิ่ม/เบี้ยปรับ, รวมยอดชำระ, วันที่ยื่น/ชำระ, เลขอ้างอิง/ใบเสร็จ

ตัวอย่างจาก `202501_PND3.pdf`:
```
แบบ ภ.ง.ด.3 | เดือนภาษี ม.ค. 2568 | ยื่นปกติ | 12 ราย
รวมเงินได้ 225,216.05 | รวมภาษีนำส่ง 6,756.48 | เงินเพิ่ม 0 | รวมชำระ 6,756.48
วันที่ยื่น 14/02/2568 | ใบเสร็จ 15/02/2568 = 6,756.48
```

### Reconcile criteria (VR-TAX-003)
ตรงระดับ: ประเภทแบบ, เดือน/ปี, จำนวนราย, รวมเงินได้, รวมภาษี, เงินเพิ่ม, รวมชำระ

## 4. เบี้ยปรับ/เงินเพิ่ม + ยื่นเพิ่มเติม (A-020)
- รองรับหลาย submission/เดือน/แบบ (ปกติ + เพิ่มเติมไม่จำกัดครั้ง)
- แยกแต่ละครั้ง: สถานะ, ครั้งที่, ยอดภาษี, เบี้ยปรับ, เงินเพิ่ม, หลักฐาน — **ห้าม overwrite ข้อมูลเดิม**
